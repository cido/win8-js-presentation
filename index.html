Jouni Kaplas from Futurice

[demo time! start menu and marco polo]

It's an IE10 without chrome

And FYI, it's a fabulous browser

Windows 8 is practically the first serious platform, which supports JavaScript apps natively.

Special global objects

  • WinJS (~50 KLOC of JS)
  • Windows (platform APIs)

Constraints

  • From operating system (eg. max memory consumption)
  • From Windows Store quality assurance process
  • From the ugly truth that JS is not (yet) truly a first class citizen

Visual Studio with on-device debugging. And the normal debugger as well.

CSS3 support in IE10

  • Regions
  • Multi-column layout
  • Flexbox
  • Transitions
  • MS-specific snap guides
  • MS-grid
    • fdsafdas

Ease of implementation

  • Appbar commands
  • Live tile
  • Sharing
  • Settings panel
  • i18n

Smooth operation and quite a lightweight footprint.

APIs are usually quite cumbersome

Lack of

  • Advanced UI elements
  • Better MVVM / MVC / what ever implementation
  • More in-depth examples and documentation
  • Testability

Scaling to different screen sizes, DPI, input devices

(sometimes even on the very same device)

Image copied from http://msdn.microsoft.com/en-us/library/windows/apps/hh780612.aspx

Things to consider

  • Not too many contenders in the app space
  • JavaScript fits very well for development of webish apps
  • Windows 8 market penetration

So...

I think Windows 8 is a platform big enough for the effort

Creating Windows 8 apps with JS is not the easiest thing out there

...but it doesn't suck too much
